Interactive demonstration of Transient, Scoped, and Singleton behaviors

üìö What is this? This demo shows how different DI lifetimes work in ASP.NET Core. Each service gets a unique ID when created, making it easy to see when instances are created vs. reused.

üîÑ Transient

New instance every time

Characteristics:

  • Created each time it's requested
  • Lightweight and stateless
  • No shared state between instances

Use Cases:

  • Utility services
  • Stateless operations
  • Default choice when unsure

üìå Scoped

One instance per request

Characteristics:

  • Created once per HTTP request
  • Shared within the same request
  • Disposed at end of request

Use Cases:

  • Database contexts
  • Unit of work patterns
  • Request-specific state

üåç Singleton

One instance forever

Characteristics:

  • Created once on first request
  • Shared across entire application
  • Lives until app shutdown

Use Cases:

  • Configuration services
  • Caching services
  • Expensive resources
